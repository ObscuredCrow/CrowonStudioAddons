Can't Go Naked

Version: 2021.100 Requirements: uMMORPG3d Script Define: No
Core Changes: Yes Tutorial Video: No

Description

This simple AddOn makes sure, that Players always have a model equipped at a equipment location that requires it. This is especially useful for designers who use "modular player models" (like Mesh Tint Tiny Heroes etc.) instead of UMA.
Example: Set "naked model" of the "head slot" to "female dark hair 1". When you now unequip an item from the "head slot", it will create "female dark hair 1" at that slots position instead. Same is true for all other equipment slots, so even with no item equipped, the model is never "naked".

Core Changes

Some add-ons require changes to the core framework in order to function. This add-on requires the following change(s):

  1. [Required] Open the following script Player.cs. (Click to Expand)
    Locate this line: 
    
[Serializable]
public struct EquipmentInfo
{
	public string requiredCategory;
	public Transform location;
	public ScriptableItemAndAmount defaultItem;
}

    Modify to match:

    
#pragma warning disable
[Serializable]
public partial struct EquipmentInfo
{
	public string requiredCategory;
	public Transform location;
	public ScriptableItemAndAmount defaultItem;
}
#pragma warning restore

Installation

  1. [Optional] Set a Model under your Player Equipment Info for the Naked Location.
