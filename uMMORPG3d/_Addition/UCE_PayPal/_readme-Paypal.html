Paypal

Version: 2021.100 Requirements: uMMORPG3d Script Define: No
Core Changes: Yes Tutorial Video: No

Description

This complex AddOn will add PayPal as payment gateway to your game. This allows you to charge your players for in game purchases with real currency. Purchases are customizable and contain coins, gold, experience as well as items. The system is relieable and fraud safe.

Core Changes

Some add-ons require changes to the core framework in order to function. This add-on requires the following change(s):

  1. [Required] Open the following script UIItemMall.cs. (Click to Expand)
    Locate this line: 
    buyButton.onClick.SetListener(() => { Application.OpenURL(buyUrl); });

    Modify to match:

    //buyButton.onClick.SetListener(() => { Application.OpenURL(buyUrl); });

Installation

  1. [Required] Add all Prefabs from Prefabs [Drag to Canvas] to your Canvas.
  2. [Required] Move all Content from Resources [Move to Resources] to your Resources folder.
  3. [Required] Create a Click Event on ButtonBuyCurrency on your ItemMall for StorePropertie SetActive True.
  4. [Required] Open a browser and go to: https://developer.paypal.com/.
  5. [Required] Click the dashboard link next to the logout button in the top right corner.
  6. [Required] Scroll down to the REST API's section of the My Apps and Credentials page and click the Create App button.
  7. [Required] Enter a name for your app select a Sandbox account then click the Create App button (You can create additional sandbox accounts by clicking on the Sandbox -> Accounts link in the left menu).
  8. [Required] In your canvas, unfold the UCE_UI_PayPal object and look for StoreProperties.
  9. [Required] You can now run and test using the sandbox (no real sale) using the buyer test email that is generated by paypal automatically (it looks like: buyer-YOURADDRESS@YOURDOMAIN.COM).

Extras

  • Using this AddOn will get your game kicked from all major app stores, as they don't tolerate third party payment providers.
  • This AddOn requires a PayPal Developer account, which is free and easy to setup.
  • By default, SANDBOX mode is enabled, you can change that before releasing a client for production. Your test email will only work while in sandbox mode.
  • By default your purchasers are forward to the google homepage after making their transaction, you can change that to another website by editing the data in: Assets\Store\Resources\Misc\CreatePaymentRequestBody.json
