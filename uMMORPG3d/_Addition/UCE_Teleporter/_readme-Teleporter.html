Teleporter

Version: 2021.100 Requirements: uMMORPG3d Script Define: _CSTELEPORTER
Core Changes: No Tutorial Video: No

Description

A greatly enhanced version of the built in teleporter with a whole lot of customization options. It uses a Collider to represent the teleporter, so you can place it everywhere in your scene. The teleporter is triggered when entered by a player and requires the player to confirm the teleport via a button click on a UI window.

Core Changes

Some add-ons require changes to the core framework in order to function. This add-on requires the following change(s):

  1. None.

Installation

  1. [Required] Add all Prefabs from Prefabs [Drag to Canvas] to your Canvas.
  2. [Required] Add all Prefabs from Prefabs [Drag to Scene] to your Scene.
  3. [Required] Add all Prefabs from Prefabs [Drag to NpcDialogue] to your Npc Dialogue under Buttons.
  4. [Required] Attach all Scripts from Scripts [Attach to NpcDialogue] to your Npc Dialogue.
  5. [Required] Link UCE_UI_Teleportation Panel and Teleportation Button to your Npc Dialogue.

Extras

  • Don't forgot to assign a teleportation target.
  • The "Simple" teleporter has no options, while the "Interactable" teleporter provides options that can limit it's interaction (requires class, item etc.).
  • In addition to the "Simple" and "Interactable" teleporters, you can also create a "Limited" Teleporter.
  • If you want to create a teleport zone with a limited player count, you have to use the objects in the "Limited Teleport" subfolder instead.
  • For a limited zone, you will need exactly one limited zone object and at least one teleporter that acts as entrance.
  • You have to link all teleporter entrances to your limited zone to make it work. Its also a good idea to corner off the zone, so it cannot be entered by normal means.
  • If you set it up correctly, you will then have a zone that only a limited number of players can enter at once - and only through the provided teleporters.
