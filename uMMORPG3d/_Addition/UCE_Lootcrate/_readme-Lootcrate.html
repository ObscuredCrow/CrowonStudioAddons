Lootcrate

Version: 2021.100 Requirements: uMMORPG3d Script Define: _CSLOOTCRATE
Core Changes: No Tutorial Video: No

Description

This AddOn allows you to create fully functional chests that can even respawn and have locks. In addition you can setup a lot of requirements for unlocking and/or accessing the lootcrate via items, skills or both. Can be used to create a simple harvesting system from it as well.

Core Changes

Some add-ons require changes to the core framework in order to function. This add-on requires the following change(s):

  1. None.

Installation

  1. [Required] Add all Prefabs from Prefabs [Drag to Canvas] to your Canvas.
  2. [Required] Add all Prefabs from Prefabs [Drag to Scene] to your Scene.

Extras

  • Interaction Requirements - are useful to set requirements for accessing the lootcrate (like skills or keys).
  • Unlock Permanently - When checked, once opened, the Interaction Requirements are skipped and any player can access the crate (even without skills or key).
  • Key - If you assign a requiredItem you will have to create those items as well (like a key or lockpick). Just use a "General" item for that.
  • Skill - If you assign a requiredSkill you need to create those skills as well and make them available to your players (like a pick lock skill). A passive skill is a good choice.
  • You can edit various settings in Interaction Requirements to make accessing the Lootcrate easier or more difficult.
  • Edit the "Animation Controller" to have a OPEN and CLOSE animation on your lootcrate. Do not rename OPEN and CLOSE (or you have to edit the code).
  • Loot Effect - is a effect game object prefab (like a particle) that is shown when a player loots the crate.
  • Loot Effect - Create a new prefab, add your particle effects, assign a Network Identity to it, add a Destroy After Component to it and add it to Registered Spawnables.
